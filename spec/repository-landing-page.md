# Specification: Repository landing page

## 1. Purpose

Define the required behaviour of the repository landing page for downstream forks and
contributors.

## 2. Scope

This specification covers the presence and minimum content of the landing page. It does
not define visual design, branding, or downstream project-specific customisation.

## 3. Requirements

- `readme-exists`: The repository MUST contain a landing page document in the agreed
  location.
- `links-to-governance`: The landing page MUST link to `AGENTS.md`.
- `explains-spec-flow`: The landing page MUST describe the specification-first
  development order used in this repository.

## 4. Risks and limitations

- Risk A: Scope may expand beyond initial onboarding notes. Mitigation: keep the first
  version minimal and factual.
